Why this Project?

"이것저것 다 해보고 싶은 호기심을 기술적 오너십으로 바꾼 첫 시도입니다." 냉장고 속 재료라는 일상의 데이터를 유저에게 유용한 가치로 전환하고, 배포 프로세스 전체를 완벽히 장악해보고자 시작했습니다.

01. Architecture

데이터의 구조를 면밀히 분석하여 무거운 RAG 대신 Filtering + Context Injection 방식을 채택했습니다. 시스템 부하를 줄이면서도 정확도를 높인 효율적인 설계를 구현했습니다.

02. DevOps

Docker와 GitHub Actions를 도입해 빌드부터 배포까지의 모든 과정을 자동화했습니다. 수동 배포의 번거로움을 해결하고 배포 시간을 20분에서 3분으로 85% 단축했습니다.

Why this Project?
